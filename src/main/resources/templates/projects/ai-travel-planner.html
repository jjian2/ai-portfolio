여행 조건을 입력하면 AI가 일정과 장소를 추천하는 맞춤형 여행 계획 서비스

01. OVERVIEW

프로젝트 개요

PROJECT GOAL

복잡한 여행 준비를 하나의 서비스로 통합

관광지 검색, 일정 작성, 지도 확인, 예산 관리와 체크리스트 작성을 각각 다른 서비스에서 처리하지 않고 하나의 웹 서비스에서 관리할 수 있도록 구성했습니다.

AI OBJECTIVE

여행 조건을 이해하는 맞춤형 일정 생성

여행지, 날짜, 인원, 예산, 여행 유형과 이동수단을 입력받아 OpenAI API가 사용자의 조건에 맞는 날짜별 여행 일정을 생성하도록 설계했습니다.

SERVICE STRUCTURE

Spring Boot와 FastAPI를 분리한 서버 구조

회원과 여행 데이터는 Spring Boot에서 관리하고, 장소 검색, 날씨 조회, RAG 검색과 AI 일정 생성은 FastAPI 서버가 담당하도록 역할을 분리했습니다.

02. ROLE

담당 역할

서비스 기획부터 화면, 백엔드와 AI 서버 연동까지 전체 개발을 진행했습니다.

사용자가 여행 조건을 입력하고, 생성된 일정을 지도와 가계부에서 실제로 활용할 수 있도록 서비스 전체 흐름을 설계했습니다.

01

Spring Boot 웹 서버와 FastAPI AI 서버를 분리하고 API 통신 구조를 설계했습니다.

02

화면 및 사용자 기능 구현

여행 일정, 장소 지도, 가계부, 체크리스트와 마이페이지 화면을 구현했습니다.

03

외부 API 연동

Google Places API, Google Maps, Weather API와 OpenAI API를 연동했습니다.

04

여행 문서를 전처리하고 임베딩하여 ChromaDB에 저장하고 검색 기능을 구현했습니다.

05. FEATURES

핵심 기능

FEATURE 01

AI 맞춤 여행 일정 생성

여행지, 기간, 인원, 이동수단, 예산과 여행 스타일을 입력하면 AI가 사용자의 조건에 맞춰 날짜별 여행 일정을 생성합니다.

  • 여행지·날짜·인원·예산 입력
  • 여행 유형과 이동수단 선택
  • OpenAI 기반 날짜별 일정 생성
  • 장소별 예상 시간과 비용 표시
  • 일정별 체크리스트 관리
FEATURE 02

방문 장소와 지도 연동

AI 일정에 포함된 장소를 Google Places API로 검색하고, 지도 마커와 장소 카드로 함께 표시합니다.

  • Google Places API v1 장소 검색
  • 장소명·주소·좌표·평점 조회
  • Google Maps 지도 마커 표시
  • 장소 이미지와 예상 비용 제공
  • Google Maps 상세 페이지 연결
FEATURE 03

여행 가계부와 예산 관리

여행 전 총예산을 설정하고, 숙소·교통·식비·쇼핑 등 카테고리별 지출을 기록할 수 있습니다.

  • 총예산 설정과 초기화
  • 지출 카테고리 추가와 삭제
  • 날짜·금액·메모별 지출 등록
  • 총지출과 남은 예산 계산
  • AI 예산 분석 기능 확장 예정
  • 영수증 OCR 기능 확장 예정
FEATURE 04

마이페이지와 회원 관리

이름, 아이디와 이메일을 확인하고 수정할 수 있으며, 계정 정보와 생성한 여행 현황도 확인할 수 있습니다.

  • 회원 정보 조회와 수정
  • 비밀번호 변경 UI
  • 가입일과 여행 개수 확인
  • 로그아웃과 회원 탈퇴
  • 생성한 여행 목록 연동 예정

06. TROUBLESHOOTING

문제 해결 경험

01

Kakao Local API 권한 문제로 장소 검색 API 변경

PROBLEM

Kakao Local API 호출 시 서버에서 500 오류가 발생했습니다.

CAUSE

Kakao Developers 프로젝트에서 필요한 추가 기능 권한을 사용할 수 없었습니다.

SOLUTION

Google Places API v1으로 전환하고 Text Search API 요청 구조를 구현했습니다.

RESULT

장소명, 주소, 좌표, 평점과 Google Maps URL을 정상 조회했습니다.

02

AI 일정 응답을 화면에서 활용하기 위한 JSON 구조 설계

PROBLEM

자연어 답변만으로는 일정과 지도를 화면에 일관되게 표시하기 어려웠습니다.

CAUSE

날짜, 시간, 장소, 비용과 좌표가 일정한 형식으로 반환되지 않았습니다.

SOLUTION

날짜별 일정, 장소, 주소, 비용과 좌표 필드를 포함한 JSON 응답을 설계했습니다.

RESULT

일정 카드, 장소 목록과 지도 마커에 동일한 데이터를 활용할 수 있게 됐습니다.
