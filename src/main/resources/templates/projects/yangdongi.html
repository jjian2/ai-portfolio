양동이

학사정보를 음성과 텍스트로 안내하는 동양미래대학교 전용 AI 챗봇

학교 규정, 학과 정보, 학사 일정과 자주 묻는 질문을 학생이 텍스트 또는 음성으로 질문하면 관련 자료를 검색해 답변하는 대학 전용 AI 챗봇입니다. Flask 기반 웹 서버와 FastAPI AI 서버를 분리하고, RAG, Whisper STT, Edge-TTS를 연결해 실제 학사 서비스 형태로 구현했습니다.

01. OVERVIEW

프로젝트 개요

어떤 서비스인가요?

대학 학사정보를 안내하는 AI 챗봇

학생이 학교 규정, 학사 일정, 시간표, 도서관, 졸업 요건 등을 질문하면 학교 데이터를 기반으로 답변하는 웹 서비스입니다.

왜 만들었나요?

흩어진 학교 정보를 빠르게 찾도록 돕기 위해

학교 홈페이지와 공지사항에 정보가 분산되어 있어 학생이 필요한 내용을 찾는 데 시간이 걸리는 문제를 하나의 대화형 서비스로 해결하고자 했습니다.

누구를 위한 서비스인가요?

동양미래대학교 재학생과 예비 학생

학사정보를 빠르게 확인하고 싶은 재학생과 학교 및 학과 정보를 찾는 예비 학생이 PC 환경에서 편리하게 사용할 수 있도록 구성했습니다.

06. TROUBLESHOOTING

문제 해결 경험

01

Flask와 FastAPI의 요청·응답 형식이 달라 통신이 실패한 문제

문제

프론트엔드에서 보낸 요청이 Flask를 거쳐 FastAPI에 전달될 때 필드명과 데이터 형식이 달라 정상 응답을 받지 못했습니다.

원인

웹 서버와 AI 서버에서 사용하는 JSON 키, 파일 전송 방식과 응답 구조가 통일되지 않았습니다.

해결

텍스트 채팅, 음성 채팅과 TTS API별로 요청·응답 규격을 정리하고 Flask 프록시에서 데이터를 변환했습니다.

결과

텍스트와 음성 요청이 동일한 UI에서 안정적으로 처리되도록 개선했습니다.

02

STT·TTS 처리 방식 차이로 응답이 지연된 문제

문제

음성 파일 업로드, STT 변환, AI 답변과 TTS 생성이 순차적으로 실행되어 사용자가 오래 기다려야 했습니다.

원인

여러 서버의 처리 단계가 많고 비동기 요청 상태를 화면에서 제대로 안내하지 않았습니다.

해결

요청 단계별 로딩 상태를 표시하고, 음성 데이터 전송과 응답 처리 흐름을 비동기 방식으로 정리했습니다.

결과

사용자가 처리 상태를 확인할 수 있고 채팅 화면이 멈춘 것처럼 보이는 현상을 줄였습니다.

03

모바일에서 사이드바와 채팅 화면이 겹치는 문제

문제

작은 화면에서 사이드바, 채팅 박스와 입력창이 겹치거나 일부 내용이 화면 밖으로 잘리는 문제가 반복되었습니다.

원인

PC 화면의 고정 좌표와 너비를 모바일에서도 동일하게 사용하고 있었습니다.

해결

모바일에서는 사이드바를 오버레이 방식으로 변경하고, 입력창과 채팅 박스의 위치와 너비를 화면 크기에 따라 계산하도록 수정했습니다.

결과

PC와 작은 화면에서 메뉴와 채팅 기능을 보다 자연스럽게 사용할 수 있게 됐습니다.

04

RAG 답변에 학교 자료의 출처가 표시되지 않은 문제

문제

장학금과 학칙을 질문해도 일반적인 답변만 제공되고, 검색된 학교 자료와 출처가 화면에 나타나지 않았습니다.

원인

문서 인덱스 상태와 검색 필터, top-k 설정이 데이터 구조와 맞지 않았습니다.

해결

문서 인덱스를 다시 생성하고 학과·데이터셋 필터와 검색 개수를 조정했으며, 답변 JSON에 출처 정보를 포함하도록 했습니다.

결과

학교 규정과 학과 데이터를 기반으로 답변하고, 참고한 자료를 함께 표시할 수 있도록 개선했습니다.
