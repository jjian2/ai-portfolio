01. OVERVIEW

프로젝트 개요

02. MY ROLE

담당 역할

Happy Whale 대회의 데이터셋 구조와 AI 기술을 조사하고, 발표 자료 기획과 발표를 담당했습니다.

단순히 기술 이름을 나열하는 방식이 아니라, 실제 대회에서 어떤 데이터가 사용되고 모델이 개체를 어떻게 구분하는지 흐름 중심으로 설명했습니다. 특히 KNN, ArcFace, 하이퍼파라미터와 MAP@5 평가 방식을 시각 자료로 구성했습니다.

07. TROUBLESHOOTING

문제 해결 경험

01

복잡한 AI 알고리즘을 짧은 발표 시간 안에 설명하기 어려웠던 문제

문제

ArcFace, KNN, MAP@5와 같은 기술은 비전공자가 한 번에 이해하기 어려웠습니다.

원인

수식과 전문 용어 중심으로 설명하면 발표 시간이 길어지고 핵심 내용이 흐려졌습니다.

해결

수식 설명을 최소화하고 그림, 사례와 비교 구조를 활용해 각 기술의 핵심 동작만 시각적으로 전달했습니다.

결과

발표 대상자가 기술의 차이와 역할을 빠르게 이해할 수 있도록 개선했습니다.

02

데이터셋의 종과 개체 ID 구조를 구분하기 어려웠던 문제

문제

species와 individual_id가 동시에 존재해 종 분류와 개체 식별의 차이를 설명하기 어려웠습니다.

원인

같은 종 안에서도 개체별 지느러미 패턴이 다르다는 데이터 특성을 먼저 이해해야 했습니다.

해결

이미지 입력 → 종 확인 → 지느러미 특징 분석 → 개체 ID 부여 순서로 데이터 흐름을 재구성했습니다.

결과

종 분류와 개체 식별의 차이를 단계적으로 설명할 수 있었습니다.

03

발표 자료의 정보량이 많아 핵심이 눈에 들어오지 않았던 문제

문제

조사한 내용을 모두 넣으면서 한 슬라이드에 글과 이미지가 과도하게 많아졌습니다.

원인

정보 전달보다 조사 내용 전체를 보여주는 데 집중했습니다.

해결

슬라이드마다 하나의 핵심 메시지만 남기고 강조 색상과 번호 구조를 사용해 시선을 정리했습니다.

결과

발표 흐름이 명확해지고 중요한 내용이 빠르게 전달되도록 개선했습니다.
