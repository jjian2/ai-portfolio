01. OVERVIEW

프로젝트 개요

어떤 서비스인가요?

자기소개서 문장의 감정과 자신감을 분석하는 AI 서비스

입력된 문장을 긍정·부정·중립으로 분류하고, 자신감 있는 표현과 부족한 표현을 함께 분석해 결과와 피드백을 제공합니다.

왜 만들었나요?

자기소개서 표현을 객관적으로 확인하도록 돕기 위해

작성자가 자신의 문장을 객관적으로 평가하기 어렵다는 점에 주목해, 감정과 자신감 표현을 시각적으로 확인할 수 있는 서비스를 구현했습니다.

누구를 위한 서비스인가요?

취업 준비생과 자기소개서를 작성하는 사용자

문장의 분위기와 자신감 표현을 점검하고, 자기소개서의 전달력을 개선하고 싶은 사용자를 대상으로 했습니다.

02. MY ROLE

담당 역할

데이터 구축부터 KoBERT 모델 학습, 웹 화면, 분석 기록과 PDF 기능까지 전체 서비스를 직접 기획하고 구현했습니다.

일반 감정 데이터셋만으로는 자기소개서 문체를 정확히 분류하기 어려워 자기소개서 문장 약 1,000개를 직접 라벨링했습니다. 이후 모델 학습, 임계값 조정, 웹 분석 기능과 결과 시각화까지 전체 흐름을 직접 구현했습니다.

06. TROUBLESHOOTING

문제 해결 경험

01

일반 감정 데이터로는 자기소개서 문장을 정확히 분류하기 어려웠던 문제

문제

NSMC와 영화 리뷰 기반 데이터로 학습했을 때 자기소개서 문장의 감정 분류 정확도가 낮았습니다.

원인

일반 리뷰 문장과 자기소개서 문장은 표현 방식과 문맥이 크게 달랐습니다.

해결

자기소개서 형태의 문장 약 1,000개를 직접 구축하고 긍정·부정·중립으로 라벨링해 재학습했습니다.

결과

실제 자기소개서 문장에 더 적합한 감정 분석 결과를 제공할 수 있게 됐습니다.

02

자신감 표현 분류가 애매한 문장에서 흔들리는 문제

문제

자신감 여부가 명확하지 않은 문장에서 결과가 기대와 다르게 분류됐습니다.

원인

Softmax 최고 확률만으로 분류하면서 문장 길이와 표현 강도를 충분히 반영하지 못했습니다.

해결

threshold를 조정하고 자신감 키워드, 문장 강도와 확률 분포를 함께 검토했습니다.

결과

명확한 문장의 분류 안정성을 높였고, 애매한 문장에 대한 한계도 파악할 수 있었습니다.

03

문서 업로드 형식별 텍스트 추출 방식이 달랐던 문제

문제

HWP와 DOCX 파일의 구조가 달라 동일한 방식으로 본문을 읽을 수 없었습니다.

원인

파일 형식마다 텍스트 저장 구조와 필요한 파서가 달랐습니다.

해결

확장자에 따라 처리 로직을 분리하고, 추출된 문장을 공통 전처리 형식으로 변환했습니다.

결과

직접 입력뿐 아니라 HWP·DOCX 문서도 동일한 분석 과정으로 처리할 수 있게 됐습니다.

04

분석 결과와 차트를 PDF로 저장하는 과정의 레이아웃 문제

문제

화면에서는 정상적으로 보이던 차트와 텍스트가 PDF에서는 잘리거나 위치가 어긋났습니다.

원인

웹 화면의 CSS 크기와 PDF 생성 시 사용하는 페이지 크기가 서로 달랐습니다.

해결

PDF 전용 크기와 여백을 정의하고 차트 이미지를 고정 비율로 변환해 배치했습니다.

결과

분석 결과와 시각화 자료를 한 문서 안에 안정적으로 저장할 수 있게 됐습니다.
