01. OVERVIEW

프로젝트 개요

어떤 서비스인가요?

URL 기반 뉴스 기사 자동 요약 서비스

사용자가 뉴스 기사 URL을 입력하면 기사 본문을 자동으로 수집하고, 자연어처리 모델이 핵심 내용을 짧은 문장으로 요약해 보여줍니다.

왜 만들었나요?

긴 뉴스 기사를 빠르게 파악하기 위해

긴 기사를 모두 읽기 어려운 사용자가 원문과 핵심 요약을 한 화면에서 확인할 수 있도록 간단한 URL 입력 방식으로 구현했습니다.

누구를 위한 서비스인가요?

짧은 시간에 뉴스 핵심을 확인하고 싶은 사용자

여러 기사를 빠르게 비교하거나 긴 기사에서 핵심 정보만 확인하고 싶은 사용자를 대상으로 했습니다.

02. MY ROLE

담당 역할

서비스 구조 설계부터 프론트엔드·백엔드 연결, 요약 모델 적용과 테스트까지 전체 과정을 직접 수행했습니다.

JSP 화면에서 입력받은 URL을 Flask API로 전달하고, Flask가 기사 수집과 모델 추론을 수행한 뒤 요약 결과를 다시 JSP 페이지에 표시하도록 구성했습니다.

06. TROUBLESHOOTING

문제 해결 경험

01

JSP와 Flask 사이에서 CORS 오류가 발생한 문제

문제

JSP에서 Flask API를 호출했지만 브라우저가 서로 다른 서버 간 요청을 차단했습니다.

원인

Java 웹 서버와 Flask 서버의 포트가 달라 교차 출처 요청으로 인식됐습니다.

해결

Flask-CORS를 설치하고 허용 출처와 요청 방식을 설정했습니다.

결과

기사 URL과 요약 결과가 정상적으로 전달되도록 개선했습니다.

02

한글 뉴스 본문이 깨져 출력된 문제

문제

기사 본문과 요약 결과 일부가 깨진 문자로 표시됐습니다.

원인

JSP와 Flask의 요청·응답 문자 인코딩 설정이 일치하지 않았습니다.

해결

양쪽의 Content-Type과 문자 인코딩을 UTF-8로 통일했습니다.

결과

한글 기사와 요약문이 정상적으로 표시됐습니다.

03

Hugging Face 모델 접근 오류가 발생한 문제

문제

기존에 사용하던 요약 모델이 비공개로 전환되면서 모델을 내려받지 못했습니다.

원인

모델 저장소의 접근 권한이 변경되어 인증 없이 사용할 수 없게 됐습니다.

해결

공개된 한국어 요약 모델을 검토하고 기본 pipeline 방식으로 교체했습니다.

결과

별도의 접근 권한 없이도 요약 서비스를 다시 실행할 수 있었습니다.
