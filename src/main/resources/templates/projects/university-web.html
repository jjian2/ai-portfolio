대학교 홈페이지
웹 서비스

신입생과 재학생이 학교 시설과 위치 정보를 직관적으로 확인할 수 있도록 만든 캠퍼스 안내 서비스

동양미래대학교의 주요 건물, 학사 서비스, 공지사항과 캠퍼스 지도를 하나의 웹사이트에서 확인할 수 있도록 구현했습니다. Java와 JSP 기반으로 전체 화면과 메뉴 구조를 설계하고, 지도 API와 SQL 연동을 통해 위치 정보와 로그인 기능을 제공했습니다.

01. OVERVIEW

프로젝트 개요

어떤 서비스인가요?

학교 시설과 학사 정보를 제공하는 캠퍼스 안내 웹사이트

주요 건물, 도서관, 학사 일정, 공지사항과 지도 정보를 하나의 화면 흐름으로 확인할 수 있도록 구현한 서비스입니다.

왜 만들었나요?

신입생이 학교 정보를 쉽게 찾도록 돕기 위해

처음 학교를 방문하는 사용자가 건물 위치와 시설 정보를 여러 페이지에서 찾지 않아도 되도록 통합된 안내 구조를 만들었습니다.

누구를 위한 서비스인가요?

신입생, 재학생과 학교 방문자

캠퍼스 시설 위치, 학사 서비스와 주변 편의시설 정보를 빠르게 확인하고 싶은 사용자를 대상으로 했습니다.

02. MY ROLE

담당 역할

팀장으로서 전체 페이지 구조와 메뉴 흐름을 설계하고, 캠퍼스 지도와 건물 소개 화면, 로그인 기능을 담당했습니다.

사용자가 학교 정보를 쉽게 탐색하도록 메인 화면과 건물 소개, 지도 페이지의 UI를 통일했습니다. 지도 API를 연동해 캠퍼스 건물과 주변 시설을 표시하고, JSP와 SQL을 연결해 로그인 세션을 구현했습니다.

01

전체 페이지 및 메뉴 구조 설계

메인 화면, 학교 소개, 건물 안내, 학사 정보와 지도 페이지의 이동 흐름과 메뉴 구조를 설계했습니다.

02

캠퍼스 메인 화면 구현

학교 전경 이미지, 검색창과 주요 메뉴를 배치해 학교 홈페이지 형태의 메인 화면을 제작했습니다.

03

건물 및 시설 소개 UI

도서관, 1호관, 2호관 등 주요 건물을 카드 형태로 구성하고 시설 설명과 상세 버튼을 제공했습니다.

04

지도 API 연동

캠퍼스 건물과 주변 음식점·카페·편의시설을 지도 마커로 확인할 수 있도록 구현했습니다.

05

사용자 정보를 데이터베이스에 저장하고 JSP 세션 기반 로그인과 페이지 접근 흐름을 구현했습니다.

06. TROUBLESHOOTING

문제 해결 경험

01

캠퍼스 건물 마커를 자동으로 생성하기 어려웠던 문제

문제

건물별 좌표 데이터를 자동으로 불러와 지도에 마커를 생성하는 과정이 제대로 동작하지 않았습니다.

원인

지도 API가 요구하는 좌표 형식과 JavaScript 데이터 구조를 정확히 맞추지 못했습니다.

해결

각 건물의 위도와 경도를 직접 조사해 정리하고, 마커 생성 코드를 반복 구조로 구성했습니다.

결과

캠퍼스와 주변 시설 위치를 지도에 안정적으로 표시할 수 있었습니다.

02

로그인 후 세션이 유지되지 않아 페이지 이동 시 로그아웃되는 문제

문제

로그인 성공 후 다른 페이지로 이동하면 사용자 로그인 정보가 사라지는 문제가 발생했습니다.

원인

HttpSession 조건 처리와 리다이렉트 흐름이 페이지마다 일관되게 적용되지 않았습니다.

해결

로그인 처리 로직과 세션 검증을 분리하고, 페이지 이동 시 동일한 세션 값을 사용하도록 수정했습니다.

결과

로그인 이후 여러 페이지를 이동해도 인증 상태가 유지됐습니다.

03

건물 카드 이미지 크기가 제각각이라 레이아웃이 흔들리는 문제

문제

건물 사진마다 비율이 달라 카드 높이와 설명 영역의 위치가 일정하지 않았습니다.

원인

원본 이미지 크기를 그대로 사용해 카드별 렌더링 높이가 달라졌습니다.

해결

이미지 영역의 높이를 고정하고 object-fit 속성과 공통 카드 스타일을 적용했습니다.

결과

건물 카드가 동일한 크기로 정렬되어 화면 완성도가 높아졌습니다.

04

기능이 많아지면서 페이지 이동 구조가 복잡해진 문제

문제

건물, 학사 일정, 공지사항과 지도 기능이 추가되면서 사용자가 원하는 정보를 찾기 어려워졌습니다.

원인

기능 중심으로 페이지를 추가해 전체 메뉴 계층과 사용자 흐름이 명확하지 않았습니다.

해결

상단 메뉴와 메인 카드 메뉴를 구분하고, 학교 소개·학사·지도 중심으로 정보 구조를 재정리했습니다.

결과

주요 기능으로 이동하는 경로가 단순해지고 사용성이 개선됐습니다.
