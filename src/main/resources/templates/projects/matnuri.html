맛누리

맛집 검색과 리뷰 분석을 결합한 AI 기반 맛집 추천 서비스

사용자가 지역 또는 음식점 이름을 입력하면 주변 맛집을 검색하고 지도에 위치를 표시합니다. 음식점의 블로그 리뷰를 수집한 뒤 긍정·부정 감정 분석과 키워드 시각화를 제공하여 사용자가 음식점의 특징을 쉽게 파악할 수 있도록 구현했습니다.

01. OVERVIEW

프로젝트 개요

어떤 서비스인가요?

맛집 검색과 리뷰 분석을 한 번에 제공하는 서비스

사용자가 원하는 지역이나 음식점 이름을 검색하면 음식점 위치와 정보를 지도에 표시하고, 블로그 리뷰를 수집해 감정 분석과 주요 키워드 시각화를 제공합니다.

왜 만들었나요?

검색 결과만으로는 알기 어려운 실제 반응을 보여주기 위해

단순한 장소 정보만으로는 음식점의 분위기나 실제 이용자의 반응을 파악하기 어렵습니다. 리뷰 내용을 분석해 긍정·부정 결과와 자주 등장하는 단어를 함께 보여주도록 구현했습니다.

누구를 위한 서비스인가요?

방문할 음식점을 빠르게 선택하고 싶은 사용자

지도 위치, 음식점 정보, 리뷰 내용과 분석 결과를 여러 사이트에서 따로 찾지 않고 한 화면에서 비교하고 싶은 사용자를 대상으로 합니다.

02. MY ROLE

담당 역할

서비스 기획부터 화면, 백엔드, 데이터베이스와 분석 기능까지 전체 과정을 직접 구현했습니다.

초기에는 Java, JSP, Servlet과 MySQL을 기반으로 맛집 검색, 지도 표시, 로그인, 관리자 기능과 리뷰 분석 기능을 구현했습니다. 이후 Spring Boot, Spring Security, JPA와 FastAPI 구조로 고도화하며 서비스 구조를 개선했습니다.

검색 화면, 맛집 추천 화면, 분석 결과 화면과 관리자 페이지의 구조와 사용자 흐름을 직접 설계했습니다.

지역 또는 음식점 이름을 기반으로 장소 데이터를 검색하고, 음식점 좌표를 지도 마커로 표시했습니다.

03

음식점 관련 블로그 리뷰 제목과 본문을 수집하고, 분석에 활용할 수 있도록 텍스트 데이터를 전처리했습니다.

04

감정 분석 및 시각화

리뷰의 긍정·부정 결과를 차트로 표시하고, 자주 언급된 단어를 워드클라우드로 시각화했습니다.

05

사용자 로그인과 세션 유지, 관리자 페이지 접근과 데이터 관리 기능을 구현했습니다.

05. FEATURES

핵심 기능

FEATURE 01

맛집 검색과 분석 요청

사용자는 지역 또는 음식점 이름을 입력해 맛집 추천 결과를 조회할 수 있습니다. 분석할 음식점 이름을 입력하면 리뷰 분석 화면으로 이동합니다.

  • 지역 또는 음식점 이름 검색
  • 맛집 추천 기능과 분석 기능 분리
  • 사용하기 쉬운 단일 화면 구성
FEATURE 02

지도 기반 맛집 추천

검색된 음식점의 위도와 경도 정보를 이용하여 지도에 마커를 표시하고, 하단 카드 목록에서 음식점 이름, 주소와 연락처를 확인할 수 있도록 구현했습니다.

  • 음식점 좌표 기반 지도 마커 표시
  • 지도와 음식점 목록 동시 제공
  • 음식점별 리뷰 분석 페이지 연결
FEATURE 03

블로그 리뷰 수집 및 목록 제공

검색한 음식점과 관련된 블로그 리뷰를 수집해 제목, 링크와 리뷰 내용을 목록으로 제공합니다. 사용자는 실제 방문 후기의 주요 내용을 한 페이지에서 확인할 수 있습니다.

  • 음식점 이름 기반 블로그 리뷰 검색
  • 리뷰 제목과 본문 미리보기 표시
  • 원문 블로그 링크 연결
FEATURE 04

리뷰 키워드 워드클라우드

수집된 리뷰에서 자주 등장하는 단어를 추출하고, 등장 빈도에 따라 크기를 다르게 표현한 워드클라우드로 시각화했습니다.

  • 불필요한 조사와 중복 단어 제거
  • 단어 등장 빈도 계산
  • 핵심 키워드 시각화
FEATURE 05

리뷰 감정 분석 차트

수집된 리뷰 문장을 긍정과 부정으로 분류하고, 각각의 리뷰 개수를 막대 차트로 표시했습니다. 음식점에 대한 전체적인 반응을 시각적으로 빠르게 확인할 수 있습니다.

  • 긍정·부정 리뷰 분류
  • 분석 결과 개수 표시
  • 막대 차트를 활용한 결과 시각화

06. TROUBLESHOOTING

문제 해결 경험

01

지도에 음식점 좌표를 자동으로 표시하기 어려웠던 문제

문제

음식점 이름만으로 지도에 정확한 위치를 표시하는 방법이 익숙하지 않아 초기에는 좌표를 직접 입력했습니다.

원인

장소 검색 API의 응답 구조와 위도·경도 데이터를 지도 마커로 변환하는 흐름에 대한 이해가 부족했습니다.

해결

장소 검색 결과의 좌표 데이터를 JSON 배열로 전달하고, JavaScript 반복문으로 마커를 생성했습니다.

결과

여러 음식점의 좌표를 자동으로 받아 지도에 반복적으로 표시할 수 있게 됐습니다.

02

지도 카드와 마커의 순서가 일치하지 않는 문제

문제

음식점 목록을 정렬한 뒤 카드를 선택하면 다른 음식점의 마커가 선택되는 현상이 발생했습니다.

원인

카드 목록은 정렬됐지만 지도 마커 배열은 기존 순서를 유지해 배열 인덱스가 서로 달라졌습니다.

해결

정렬된 음식점 데이터를 기준으로 마커를 다시 생성하고, 카드와 마커에 동일한 식별자를 적용했습니다.

결과

카드 선택 시 해당 음식점의 마커와 지도 위치가 정확하게 연결됐습니다.

03

로그인 후 세션이 유지되지 않던 문제

문제

로그인에 성공해도 다른 페이지로 이동하면 사용자 정보가 사라지거나 다시 로그인해야 하는 문제가 있었습니다.

원인

HttpSession 저장 시점과 페이지 이동 처리 순서가 정확하게 구성되지 않았습니다.

해결

로그인 성공 시 사용자 정보를 세션에 저장하고, 페이지 이동 전에 세션 상태를 검사하도록 로직을 수정했습니다.

결과

로그인 사용자의 정보가 여러 페이지에서 안정적으로 유지되도록 개선했습니다.

04

Spring Boot와 FastAPI 서버 간 통신 문제

문제

Spring Boot에서 FastAPI 분석 서버를 호출할 때 요청이 실패하거나 한글 데이터가 정상적으로 전달되지 않았습니다.

원인

서버 간 요청 형식과 Content-Type, CORS와 문자 인코딩 설정이 서로 일치하지 않았습니다.

해결

요청과 응답 DTO를 일치시키고 UTF-8 인코딩과 CORS 설정을 추가해 통신 구조를 정리했습니다.

결과

음식점 이름과 리뷰 데이터가 정상 전달되고, 분석 결과를 웹 화면에 표시할 수 있게 됐습니다.
